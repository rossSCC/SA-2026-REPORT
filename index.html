MEETING THE PROJECT BRIEF

Leaving Certificate Computer Science – Project Brief - 2026

Forests are some of the most important ecosystems on our planet. They give us clean air, store huge amounts of carbon, protect biodiversity, and help regulate water and temperature. But forests are under threat from climate change, wildfires, and biodiversity loss. As trees face droughts, higher temperatures, and diseases, their ability to support wildlife and store carbon is reduced.

https://commons.wikimedia.org/wiki/File:Pine_trees_in_Wicklow_Mountains.jpg

Computer science provides tools to help us understand and respond to these challenges. By creating embedded systems, we can collect environmental data such as soil moisture, light, or temperature, and simulate real-world processes like canopy cover, drought cycles, or wildfire risk. We can also build computer-based models that test “what-if” scenarios, compare different outcomes, and make predictions about the future of forests, climate, biodiversity, and species populations. These insights help us to understand risks, anticipate future impacts, and explore possible solutions.

The Task



For this project you are required to develop a functional embedded system that collects environmental data and simulates a real-world process related to the theme of forests and their influence on climate, biodiversity, water resources, species populations, carbon storage, wildfire risk, etc.
You will also develop a computer-based model that uses the collected data that can be combined with open-source data or simulated data (dataset created by you), to test “what-if” scenarios, carry out predictive analysis about future conditions, and compare outcomes. Finally, you will incorporate a feedback mechanism to make the system adaptive, allowing either your embedded system or your model to respond automatically to changing conditions.

RESEARCH AND INVESTIGATION

The Problem:



  • In Ireland, 31% of fires recorded by satellite imaging programs are in areas internationally recognised for their role in conserving species and ecosystems. This represents a major loss to biodiversity and forest health.
  • Additionally, there is a considerable risk to hikers and campers. For example, over 25,000 people climbed Croagh Patrick on Reek Sunday last year. A significant wildfire on that day could put thousands at risk and add a tremendous strain on the Search and Rescue services.
  • Currently, in Ireland there is no warning system in place in our forest parks to warn any hikers or campers about the current danger levels. In other countries such as America, Germany, etc it is common to see an analogue sign warning people of the current risk of a wildfire.

Existing Solutions:



  • A project from University College Cork called FLAREs (Fires, Land and Atmospheric Remote Sensing of EmissionS) “aims to develop systematic approaches to the acquisition and collation of a range of data on agricultural and uncontrolled wildland burning burn events from satellite datasets.” This data could be used to feed into a warning system however it has some issues, for example,
    • Each overpass by the satellite will record every fire as a new fire creating duplicates.
    • The project was from 2015 to 2021 and was extended to 2024, currently it is not running.
  • The Met Éireann website displays a map with any current weather warnings, this is a workable solution however it has some limitations, primarily this website does not work where there is no mobile reception.
    The website only displays a low resolution of warnings, fires are often localised when they start and the risk of a fire breaking out can change drastically from one place to another.

Additional Research



  • I used the ComReg (organisation for the regulation of the communications sector) mobile data coverage map to estimate the phone coverage over Wicklow Mountain National Park.
  • I studied 4th Generation signal (4G) by Virgin Media, from the map I concluded that coverage is generally weak and patchy across the mountain range, with noticeable improvement near main roads and towns.

Project options



Since I have identified forest fires as a severe disaster, I need to identify what environmental variables I need to measure to model the risk of a forest fire.

Variable Correlation Advantages Disadvantages
Temperature and Humidity Elevated temperatures combined with low humidity create dry conditions that make vegetation highly flammable. This project would require an added sensor which would add to the project. While there is a correlation between humidity and vegetation moisture, Ireland is a humid country meaning the difference between high risk and low risk is slim leading to a percentage error effect.
Temperature and Light intensity A hot day dries out vegetation and intense light gives the energy needed to start a fire. Simplicity of design, both variables can be measured on the micro bit. I would need to add something to my embedded system. So, it is not too simple.
Light intensity and wind speed High brightness means increased likelihood of fire, and strong wind means even a little fire will grow rapidly. Wind adds a unique element of fire spread. Only one variable to calculate fire risk.

Justification
Measuring temperature and light intensity is the most logical choice to me, it allows me to natively collect data which should be more reliable and easier to implement. This approach has a clear correlation between environment and fire risk. The downside is easily mitigated by adding an added display or output device to convey risk.

    Software:
    • Micro:bit runs on micro python; this has the same syntax as regular python.
    • I began with creating a flowchart for the operations of the Microbit before moving to developing the final program.


  2. Simulation side:
    I realized that relying only on two variables lacked complexity and could cause false positives. For instance, if a day is hot and bright but experienced heavy morning rain, my local sensors would incorrectly report a high fire danger. To solve this, I designed a hybrid system integrating the Met Eireann weather API to provide rainfall amounts each day.

    System Flow:
    The Simulation model can be understood from the system diagram. We see the microbit feeds in the sensor data, and the API returns its CSV on request.
    The program flow is controlled by the main menu script, this handles tasks and data transfer between algorithms.
    I broke the program into two other files, one to handle the incoming data by ensuring it exists and is in a usable format.
    The other simulation script is what calculates the risk for each day and infers meaningful data from the three data sources.

Problems encountered


- Met Eireann API integration.

The problem:

  • During integration I initially tried to read the first line of the microbit data and the API data using Pandas to get the column headers. While the microbit returned the expected result, reading the “rain” column of the API data returned a missing column error.

                        /home/runner/workspace/data_manager.py:67: UserWarning: column ‘rain’ not found

Steps taken:
  • To find the cause of this error I opened the API URL in my browser, this downloaded the CSV file so I could view it.
  • When I opened the file, I found that the API returns a header of 10 rows containing meta data such as station name, height above sea level, location and more. I also found that simply skipping this header was not sufficient to fix this issue.
  • The Met Eireann API retuns data in a historical format where the newest entry is last in the file. The problem with this is that unlike the microbit data, this file has rainfall measurements from the 1st of May 1965! That means to find the relevant data I would have to read the file backwards as simply going to a fixed position would slowly drift away from the current date.
  • To resolve this, I used a Pandas function to get the last few rows, equal to the length of the Micro:bit data. 

                        recent_rain = float(weather.tail(len(micro))['rain'])

  • And to deal with the Meta-data header I modified the read_csv() parameters to explicitly skip the first 10 rows (skiprows=10)
