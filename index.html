Project Summary: Forest Sentinel

This project focuses on the Forests theme by creating a **Smart Forest Sentinel**—an embedded system designed for autonomous wildfire risk prediction. It meets all basic requirements by implementing a standalone Micro:bit system and an offline data logging method.

Advanced Requirement Coverage

The system hits advanced requirements through:

  • **Adaptive System:** The "Dry Day Counter" algorithm increases the risk score based on the cumulative effect of sustained dry conditions, accurately simulating real-world fire fuel readiness.
  • **Disaster Risk Modelling:** The Python component runs a predictive model that combines sensor data with historical factors to calculate a Fire Weather Index.
  • **What-If Scenarios:** Two distinct scenarios simulate external factors (e.g., climate change and deforestation) to demonstrate risk extrapolation.

// 2. Investigation & Research

Initial investigation focused on the scientific principles behind fire danger rating systems, specifically the **Cumulative Effects** of weather. This informed the decision to move beyond simple temperature thresholds and integrate a running counter for dry periods.

Sensor Capabilities

I chose the Micro:bit's internal sensors because they simplify the hardware (fewer components, less potential failure) and still provide the required data points: **Temperature** (for thermal fuel activation) and **Light Level** (as a proxy for solar radiation/drying).

// 3. Plan & Design

System Architecture

The system uses a 'Store and Dump' architecture. The Field Unit (Micro:bit) operates autonomously, saving data to its flash storage. The Analysis Unit (Python) retrieves this data via serial connection for complex modelling.

// 4. Create & Development

The 24-LED Halo Output

The 24 LEDs are divided to visualize risk: LEDs 1-8 are Green (Low), 9-16 are Yellow (Moderate), and 17-24 are Red (High/Critical). This provides a non-textual, easily understood visual cue.

The Calibration Solution

**Problem:** Risk calculations were inconsistent between indoor and outdoor testing. **Solution:** Button A was programmed to set a `Baseline_Light` on boot. The risk calculation now uses a ratio: `Current_Light / Baseline_Light`, making the system adaptable to different environments.

Python Modelling: Running the Scenarios

The Python script uses the Pandas library to load the data from the CSV file. The scenario logic is implemented by manipulating the temperature column before re-running the risk calculation.

// 5. Evaluation & Conclusion

Evaluation of Final Artefact

The project successfully demonstrates a clear understanding of embedded system design and computational modelling. The adaptive algorithm proved effective in generating higher risk scores after sustained dry periods, validating the simulation.

Future Improvements

To further enhance the project, I would integrate a more robust data transmission protocol, such as LoRa, to enable true real-time remote monitoring without the need for manual data dumping. Additionally, incorporating a dedicated **Humidity Sensor** would make the FWI prediction model more accurate.
